# 当日運営フロー（Runbook）

## 事前（前日〜当日朝）

1. `.env` / Vercel 環境変数が正しいか確認
2. Supabase Realtime が有効か確認
3. 表示端末でキオスク URL をブックマーク（[KIOSK_SETUP.md](./KIOSK_SETUP.md)）
4. `/rehearsal` で通し確認（任意）

## 当日フロー

### 1. 表示端末起動

```
/display?eventId=<UUID>&kiosk=1
```

ヘッダーが **接続済み** になるまで待つ。

### 2. 運営端末（/admin）

1. パスコード入力
2. **参加者登録** — カメラで 8 名（名前 + レーティング）
3. **抽選** — プレビュー確認 → ブラケット生成
4. Display に WB/LB/GF 表が表示されることを確認

### 3. 試合進行

1. **試合進行** タブ — 現在試合の勝者をタップ
2. Display で演出（線 → 衝突 → 爆発 → VS）
3. 急ぎのときは **演出スキップ**
4. 誤操作時は **1 手戻す**（undo）

### 4. 大会終了

1. 全試合完走 → 優勝表示
2. **終了** タブ — 「削除する」と入力 → **大会終了・肖像データ削除**
3. Storage / DB から顔写真が削除される（不可逆）

## リハーサル（本番前）

1. `/rehearsal` → **リハーサル開始**
2. 完了後 **Display を開く** — 別タブで演出確認
3. 本番 event は `[REHEARSAL]` 名前のイベントと分離される

## 連絡・エスカレーション

- Realtime 切断: Display が自動再接続。30 秒以上 **接続済み** に戻らない場合は Wi-Fi 確認
- 演出が出ない: Admin で勝敗確定済みか、Display の eventId が一致しているか確認
- ブラケットが古い形式: Admin 抽選タブで再生成
